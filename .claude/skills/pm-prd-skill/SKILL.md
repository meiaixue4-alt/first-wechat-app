# PM PRD Skill

这是一个纯执行型 skill，由 `chnapinjingli` 在阶段 3 调度。只做一件事：根据上游确认的需求，生成极简 MVP PRD 并落盘。

## 输入要求

调用此 skill 前，上游（chnapinjingli）必须已经确认：
- 项目名称（或简称）
- MVP 核心场景（3-5 个）
- 明确的范围边界（做什么、不做什么）
- 目标用户画像

## 输出规范

生成一个 Markdown 文件，结构如下：

```markdown
# {项目名} MVP PRD

## 1. 项目概述
- 一句话描述项目是什么
- 目标用户是谁
- 核心价值主张

## 2. MVP 范围
### 本期做
- [核心功能 1]
- [核心功能 2]
- [核心功能 3]

### 本期不做
- [明确排除的功能 1]
- [明确排除的功能 2]

## 3. 核心 User Story
| ID | 用户角色 | 场景 | 期望结果 | 优先级 |
|----|---------|------|---------|--------|
| US-01 | ... | ... | ... | P0 |
| US-02 | ... | ... | ... | P1 |

## 4. 验收标准
- [可量化的成功指标 1]
- [可量化的成功指标 2]
```

## 文件保存

- 路径：`docs/{项目名}/PRD_{项目名或简写}.md`
- 若目录不存在，先创建 `docs/{项目名}/`

## 核心守则
- **极简**：只写 MVP 必需的内容，不扩充、不脑补
- **不画图**：不生成架构图、时序图、流程图
- **可测试**：每条 User Story 必须可验收
