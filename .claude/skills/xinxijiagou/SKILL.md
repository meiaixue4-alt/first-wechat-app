## 阶段 1：发现 (Discovery)

目标：理解网站的核心目标、目标用户和"北极星指标"。

1. **确定背景**：询问用户网站的主要用途（例如：销售、展示、工具等）。
2. **分析用户**：谁是访客？（例如：买家、学生、管理员等）。
3. **寻找北极星行动**：如果访客只能做一件事，那应该是什么？
4. **限制提问**：每轮对话最多提 2 个问题，保持简洁。
5. **给出建议**：基于用户回答，主动给出你的理解和建议，让用户反馈。

## 阶段 2：架构 (Architecture)

目标：共创站点地图 (Sitemap) 和用户路径，识别核心页面。

1. **提议站点地图**：不要等待用户发明结构，基于第一阶段的信息，主动给出一个页面列表。
2. **确认用户路径**：模拟用户如何从进入首页到完成"北极星行动"。
3. **识别核心页面**：共同选出 3-5 个最重要的页面（通常包括首页、转化页等）。
4. **规划内容模块**：为核心页面列出主要的内容块（如：Hero 区、特性列表、定价、底部等）。

## 阶段 3：线框图确认 (Wireframe Confirmation)

目标：为核心页面生成直观的 ASCII 线框图，帮助用户在脑海中建立具体的页面结构认知。

1. **生成线框图**：通过 ASCII 线框图，展示核心页面的大致布局和内容分布。
2. **请求确认**：询问用户这个布局是否符合预期，并收集修改意见。

## 阶段 4：交付 (Delivery)

目标：在用户确认后，销毁具象的线框图，生成对前端设计 Agent 极其友好且无束缚的结构交付物。

1. **销毁线框图**：在最终生成的 IA 文档中**绝对禁止包含任何阶段 3 的 ASCII 几何物理排版线框图**。必须将最终版面布局的创造权与自由度完全留给后续的设计生成 Agent。
2. **生成 Sitemap 图**：使用 Mermaid `graph TD` 语法生成网站地图。
3. **构建内容模块清单 (Content Modules)**：将阶段 3 确认的布局转化为纯文本的高密度数据要求与内容元件清单（如：Hero 区必须包含主标题、副标题、以及 1 个 CTA 按钮，但不指定具体排列方式）。
4. **汇总 IA 文档**：创建一个包含项目概述、Sitemap、用户路径流向、高密度核心内容模块清单的 Markdown 文档。
5. **命名建议**：`{项目名}-ia-{YYYY-MM-DD}.md`，并保存至 `docs/{项目名}/`。

## 核心守则
- 始终通过提问和主动反馈来保持协作。
- 引导用户收敛，而不是无止境的发散。
- 如果用户已经有想法，可以直接跳过发现阶段，进入架构或交付阶段。
