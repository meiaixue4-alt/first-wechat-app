# 格物致知 MVP PRD

> **版本**：v1.0
> **日期**：2026-05-21
> **类型**：微信小程序

---

## 1. 项目概述

### 一句话描述
每日推送 5 条有科学依据的生活消费小知识，帮普通人减少选错商品、被谣言误导、花冤枉钱的情况。

### 目标用户
- 没有系统理工科背景的普通消费者
- 对日用品选购有困惑、容易被营销话术带偏的人
- 愿意把实用知识转发给家人的"家庭科普员"

### 核心价值主张
用一句话 + 一张图 + 一段解释，让非专业人士 30 秒内学会一个"不再被坑"的实用判断标准。每条内容有据可查，接受用户质疑和专家纠错。

## 2. MVP 范围

### 本期做

| # | 功能 | 说明 |
|---|------|------|
| F1 | 首页「今日」5 条卡片 | 五大分类各推送一条当日 tip，纵向排列滑动浏览 |
| F2 | Tips 详情页 | 点击卡片进入，展示完整正文 + 参考文献 + 互动操作 |
| F3 | 反馈三板斧 | 每条 tip 支持"有用""不感兴趣""质疑"，质疑弹窗含身份标注 |
| F4 | 微信原生转发 | 使用 `<button open-type="share">` 实现分享，分享标题为 tip 一句话结论 |
| F5 | 收藏/取消收藏 | 详情页及卡片上 toggle 收藏，同步至「我的收藏」列表 |
| F6 | 「发现」长廊 | 按分类筛选 + 关键词搜索往期所有 tip，日期倒序，分页加载 |
| F7 | 「我的」页面 | 微信头像/昵称展示 + 收藏列表 + 浏览历史 + 质疑反馈记录 + 设置入口 |
| F8 | 底部三 Tab 导航 | 今日 / 发现 / 我的，使用 `app.json` 原生 `tabBar` |
| F9 | 分享落地页引流 | 详情页底部展示"去看看今天的 Tips"入口，引导裂变用户进入首页 |

### 本期不做

| # | 功能 | 原因 |
|---|------|------|
| N1 | CMS 内容管理后台 | MVP 阶段内容量小，使用静态数据/本地 JSON 硬编码 |
| N2 | 运营审核后台（质疑处理） | 质疑反馈仅落盘存储，人工通过云开发数据库查看，不做审核界面 |
| N3 | 模板消息推送（每日提醒） | 涉及订阅消息申请和审核，MVP 后评估 |
| N4 | 个性化推荐/算法排序 | MVP 固定每日 5 条，不根据用户反馈做动态调整 |
| N5 | 用户注册/登录系统 | 仅使用 `wx.login` 获取 openid 做匿名标识，不要求手机号或注册 |
| N6 | 评论区/Tips 讨论 | MVP 聚焦单向内容消费 + 反馈，不做社区化 |
| N7 | 查看他人有用数/质疑内容 | MVP 仅展示有用累计数，不公开质疑内容，避免争议扩散 |

## 3. 核心 User Story

| ID | 用户角色 | 场景 | 期望结果 | 优先级 |
|----|---------|------|---------|--------|
| US-01 | 普通用户 | 每天早上打开小程序，看一眼今天的生存技巧 | 首页展示 5 张图文卡片（五分类各一），30 秒内读完一条，无需登录或操作即可浏览 | P0 |
| US-02 | 普通用户 | 觉得某条 tip 有用，想转发给爸妈 | 点击卡片上的转发按钮，调起微信分享，分享卡片标题为 tip 的一句话结论 | P0 |
| US-03 | 普通用户 | 被朋友分享的 tip 吸引，点进来看看 | 打开分享链接进入该 tip 详情页，读完正文和依据，底部有入口引导进入小程序首页看更多 | P0 |
| US-04 | 普通用户 | 想查看往期某一个分类的所有 tips | 切换到「发现」Tab，点击分类筛选（如"护肤品真言"），列表展示该分类所有往期内容，支持上拉加载更多 | P1 |
| US-05 | 普通用户 | 用搜索栏找某条之前看过的 tip | 在「发现」页搜索框输入关键词，返回标题/正文匹配的 tip 列表 | P1 |
| US-06 | 普通用户 | 收藏 tips 方便购物时查阅 | 在详情页点击收藏按钮，tip 加入「我的收藏」列表，购物时打开查看判断标准 | P1 |
| US-07 | 专业用户 | 发现某条 tip 存在事实错误，想指正 | 点击"质疑"按钮，弹出表单，填写质疑理由和可选参考文献，标注自己身份后提交 | P1 |
| US-08 | 普通用户 | 对某类内容不感兴趣，希望少看到 | 点击卡片上的"不感兴趣"按钮，记录反馈（MVP 仅记录，不做排序调整） | P2 |
| US-09 | 普通用户 | 点"有用"表示认可，也看到别人觉得有用 | 点击"有用"按钮，累计数 +1 并展示在详情页，同时记录反馈 | P2 |
| US-10 | 普通用户 | 查看自己提交过的质疑反馈 | 在「我的」→「我的质疑反馈」中看到历史提交记录 | P2 |

## 4. 验收标准

### 4.1 功能完整性

- [ ] 首页加载并展示当日 5 条 tip 卡片（五分类各一），卡片含分类标签、主视觉图、标题、正文摘要
- [ ] 点击卡片进入对应详情页，展示完整正文、参考文献、互动按钮
- [ ] 详情页转发按钮触发微信原生分享，分享标题为 tip 的一句话结论
- [ ] 详情页底部展示引流入口"去看看今天的 Tips"，点击跳转首页
- [ ] 收藏按钮 toggle 正常，收藏状态同步至「我的收藏」列表
- [ ] 「发现」页支持按 5 个分类筛选，列表按日期倒序，上拉加载更多
- [ ] 「发现」页搜索框输入关键词后返回匹配结果
- [ ] 「我的」页展示微信头像/昵称、收藏数、历史、质疑反馈数、设置入口
- [ ] 质疑弹窗包含：多行文本输入（必填）、附件上传入口（选填）、身份单选（选填），提交后关闭弹窗并提示
- [ ] 有用/不感兴趣按钮点击后记录反馈，有用累计数在详情页可见
- [ ] 底部三 Tab 导航切换正常，当前 Tab 高亮

### 4.2 微信小程序规范

- [ ] 所有页面在 `app.json` 中注册，底部 Tab 使用 `app.json` 的 `tabBar` 配置
- [ ] 全部 API 调用使用 `wx.*` 原生接口，无浏览器端 API
- [ ] 页面文件为 WXML + WXSS + JS + JSON 四件套，无第三方前端框架
- [ ] 转发使用 `<button open-type="share">`，不自行实现分享逻辑

### 4.3 内容与体验

- [ ] 每条 tip 卡片的一句话结论 ≤ 15 字
- [ ] 每条 tip 正文含可查证的参考文献来源（国标号/论文/权威机构）
- [ ] 内容面向非专业读者，无未解释的专业术语
- [ ] 质疑入口在卡片和详情页均可见且可达

### 4.4 数据与性能

- [ ] 首页 5 条卡片首次渲染时间 < 1.5 秒（模拟器环境）
- [ ] 发现页列表分页加载，每页 20 条
- [ ] 收藏、有用、不感兴趣、质疑等操作均有本地存储兜底（网络失败不影响操作）
- [ ] `wx.login` 获取 openid 失败时不影响核心浏览功能

## 6. 数据模型（MVP 最小集）

### Tip
```json
{
  "id": "string",
  "category": "material | food | skincare | agri-industry | health",
  "date": "2026-05-21",
  "title": "一句话结论（≤15字）",
  "summary": "卡片上的2-3句简短解释",
  "body": "详情页完整正文（富文本/多段落）",
  "image": "/images/tips/xxx.png",
  "references": ["GB 4806.7-2016", "来源2"],
  "usefulCount": 128,
  "createdAt": "ISO datetime"
}
```

### UserFeedback
```json
{
  "id": "string",
  "tipId": "string",
  "userId": "openid",
  "type": "useful | not-interested | dispute",
  "disputeReason": "质疑理由文本（仅 type=dispute 时）",
  "disputeAttachment": "图片/链接（可选）",
  "identity": "student | practitioner | researcher | layperson | null",
  "createdAt": "ISO datetime"
}
```

### UserFavorite
```json
{
  "userId": "openid",
  "tipId": "string",
  "createdAt": "ISO datetime"
}
```
